The documentation within this wiki uses coloring to differ between different type of information. Below, these annotations and colors are described.

Command Line

If you encounter something like this:

netconvert --visum=MyVisumNet.inp --output-file=MySUMONet.net.xml

you should know that this is a call on the command line. There may be also a ‘\’ at the end of a line. This indicates that you have to continue typing without pressing return (ignoring both the ‘\’ and the following newline). The following example means exactly the same as the one above:

netconvert --visum=MyVisumNet.inp \   --output-file=MySUMONet.net.xml

Referenced Data Types

  • <BOOL>: a boolean value, use “t” or “true” and “f” or “false” for encoding
  • <INT>: an integer value, may be negative
  • **: an unsigned integer value, must be >=0
  • **: a floating point number
  • **: time, given in seconds; fractions are allowed, e.g. “12.1”
  • **: any string, but use ASCII-characters only

  • **: a string which must not contain the following characters: '\#'

  • ** or **: the (relative or absolute) path to a file; see also [\#Referenced File Types](/#Referenced_File_Types "wikilink")
  • **: a (a relative or absolute) path (usually to a folder)
  • **: a quadruple of floats separated by ',' (**,**,**,**), which describe the red, green, blue, and alpha component ranging from 0.0 to 1.0 (the alpha component is optional), alternatively the list may contain integers in the 0-255 range. Please note that the separator must be a comma and there are no spaces allowed. The color may also be defined using a single string with a [HTML color code](http://en.wikipedia.org/wiki/Web_colors#Hex_triplet) or one of the basic colors (“red”, “green”, “blue”, “yellow”, “cyan”, “magenta”, “black”, “white”, “grey”)
  • <2D-POSITION>: two floats separated by ‘,’ (**,**), which describe the x- and the y-offset, respectively. z is 0 implicitly
  • <3D-POSITION>: three floats separated by ‘,’ (**,**,**), which describe the x- , y- and the z-offset, respectively
  • **: A list of 2D- or 3D-Positions separate by ' '. I.e. (*<2D-POSITION>* *<2D-POSITION>*,*<3D-POSITION>*)
  • <2D-BOUNDING_BOX>: four floats separated by ‘,’ (**,**,**,**), which describe x-minimum, y-minimum, x-maximum, and y-maximum
  • **: a string containing the projection definition as used by proj.4; please note that you have to embed the definition string in quotes

Referenced File Types

  • **: a [SUMO network file](/Networks/SUMO_Road_Networks "wikilink") as built by [NETGENERATE](/NETGENERATE "wikilink") or [NETCONVERT](/NETCONVERT "wikilink")
  • **: a [SUMO routes file](/Definition_of_Vehicles,_Vehicle_Types,_and_Routes "wikilink") as built by [DUAROUTER](/DUAROUTER "wikilink") or [JTRROUTER](/JTRROUTER "wikilink") or by hand
  • **: a [SUMO edge type file](/SUMO_edge_type_file "wikilink"), built by hand or downloaded
  • **: a [OpenStreetMap file](/OpenStreetMap_file "wikilink") as exported from [OpenStreetMap](http://www.openstreetmap.org/)
